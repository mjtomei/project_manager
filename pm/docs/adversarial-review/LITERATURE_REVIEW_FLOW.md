# Literature Review Flow

The top-level pipeline for producing a literature review using this repo's methodology. Read this first; the per-phase detail files follow.

## Phases

### Phase 1 — First draft

A literature review draft is generated by an LLM (or a human + LLM collaborator) covering:
- The surrounding literature, organized by topic or sub-cluster.
- The closest published peers for each claim the artifact wants to make.
- The artifact's positioning relative to that landscape — where it inherits, where it diverges, where it is genuinely first.

The draft is intentionally rough; it provides the structural skeleton subsequent phases refine. Don't perfect it before moving on — the iterative phases catch what's missing more cheaply than a careful first draft does.

### Phase 2 — Iterative adversarial review cycles (mostly automatic)

Detail: `METHODOLOGY.md`.

A blind, fresh reviewer audits the draft against the four review blocks (substance, structure, accessibility, prose). The author writes a response file (agree / partially-agree / disagree, with reasoning), applies the accepted edits, and a fresh blind reviewer starts the next cycle. Each cycle includes:

- A citation-graph walk (METHODOLOGY step 5) to find missed prior art — the structural complement to Phase 3's audit. The walk finds what *should* be cited; the audit (Phase 3) checks that what *is* cited is used faithfully.
- A whole-document verbosity pass (step 11).
- A net-cut once length is flagged (step 10).
- A light citation-use sanity check (step 6, which points to `CITATION_USE_AUDIT.md` for the thorough version).

Cycles continue until findings get pedantic — METHODOLOGY's convergence signal (step 9).

This phase is *mostly automatic*: each blind reviewer pass runs as a background agent against the methodology prompts; the author's interaction surface is the response cycle and the apply pass.

### Phase 3 — Detailed per-citation audit

Detail: `CITATION_USE_AUDIT.md`.

Once iterative cycles converge, the dedicated full-text citation audit runs. Every load-bearing citation is read in full (not just the abstract), checked against the artifact's characterization, and any over- / under-characterization or unrepresented alternative perspective surfaces as a proposed substantive rewrite in a standalone audit doc. The author walks the audit doc line by line and applies the rewrites.

This phase catches the over-characterization failure mode that lit-search agents and reviewer summaries routinely introduce — and that slips past blind reviewers because reviewers check against the artifact's text rather than the source's actual content.

### End state

A literature review whose substance has been adversarially probed across multiple blind cycles, whose structure and prose have been refined, and whose citations have been individually verified against full-text sources with any over-characterization corrected.

## Iteration between phases

The flow is linear in conception, but the citation audit (Phase 3) can surface issues that warrant a short follow-up revision cycle. That is expected — the audit is both the final pass *and* a feedback mechanism. Once the audit's proposed rewrites apply cleanly without surfacing further substantive issues, the literature review is complete.

If the audit surfaces *substantial* substantive issues (e.g., a load-bearing citation turns out to mean something materially different from the doc's claim, undermining a contribution claim), the right move is to return to Phase 2 for a focused cycle that addresses the new substantive problem — *not* to patch with a hedge and ship.

## Companion files

- `METHODOLOGY.md` — the adversarial-review-cycle protocol (Phase 2).
- `CITATION_USE_AUDIT.md` — the detailed citation audit (Phase 3).
- Per-cycle artifacts in `pm/docs/adversarial-review/`:
  - `REVIEW_CYCLE_<n>_<artifact>.md` — verbatim findings from each blind reviewer.
  - `REVIEW_RESPONSE_CYCLE_<n>_<artifact>.md` — the author's response.
  - `CITATION_AUDIT_<artifact>_<context>.md` — standalone audit doc from Phase 3.
